# 🚀 Velum - Запуск и управление

> **Единый скрипт управления:** `velum.sh`
>
> **База данных по умолчанию:** PostgreSQL (SQLite устарел)

---

## 📋 Быстрый старт

### Вариант 1: Dev (PostgreSQL в Docker) - рекомендуется для разработки

```bash
# Инициализация (запуск PostgreSQL + создание БД и админа)
./velum.sh init dev

# Запуск
./velum.sh start dev

# Остановка
./velum.sh stop
```

**Доступ:** http://localhost:3000  
**Логин/пароль:** `admin / admin123`

---

### Вариант 2: Docker (всё в Docker) - для продакшена

```bash
# Запуск всех сервисов в Docker
./velum.sh start docker

# Остановка
./velum.sh stop
```

**Доступ:** http://localhost  
**Логин/пароль:** `admin / demo123`

---

## 📖 Все команды

| Команда | Описание |
|---------|----------|
| `./velum.sh start [dev\|docker]` | Запуск сервиса в выбранном режиме |
| `./velum.sh stop` | Остановка всех сервисов |
| `./velum.sh restart` | Перезапуск сервисов |
| `./velum.sh clean` | Очистка данных (БД, volumes) |
| `./velum.sh init dev` | Инициализация БД (миграции + создание админа) |
| `./velum.sh status` | Показать статус сервисов |
| `./velum.sh logs` | Показать логи в реальном времени |
| `./velum.sh build` | Сборка проекта (backend + frontend) |
| `./velum.sh demo [dev]` | Запуск с демо-данными |
| `./velum.sh help` | Показать справку |

## 🎯 Режимы запуска

### Dev (PostgreSQL в Docker)

**Преимущества:**
- ✅ PostgreSQL в изолированном контейнере
- ✅ Backend работает на хосте (быстрая отладка)
- ✅ Рекомендуется для разработки
- ✅ Полная совместимость с продакшеном

**Требования:**
- Docker и Docker Compose
- Rust/Cargo
- Опционально: Node.js (для сборки frontend)

**Расположение данных:**
- БД: Docker volume `velum_postgres_data`
- Логи: `./logs/backend.log`

---

### Docker (всё в Docker)

**Преимущества:**
- ✅ Полная изоляция
- ✅ Все сервисы в контейнерах
- ✅ Готово для продакшена

**Требования:**
- Docker и Docker Compose

**Расположение данных:**
- БД: Docker volume
- Все сервисы управляются через docker-compose

## 🔍 Диагностика проблем

### Backend не запускается

1. Проверьте логи:
   ```bash
   ./velum.sh logs
   ```

2. Проверьте зависимости:
   ```bash
   cargo --version
   docker --version
   ```

3. Попробуйте другой режим:
   ```bash
   ./velum.sh start docker  # если dev не работает
   ```

### PostgreSQL не запускается (dev/docker)

1. Проверьте Docker:
   ```bash
   docker ps | grep velum-db
   ```

2. Перезапустите контейнер:
   ```bash
   docker restart velum-db
   ```

3. Проверьте логи PostgreSQL:
   ```bash
   docker logs velum-db
   ```

### Frontend не отображается

1. Проверьте наличие файлов:
   ```bash
   ls -la web/public/
   ```

2. Проверьте, что marketplace.html существует:
   ```bash
   ls -la web/public/marketplace.html
   ```

3. Перезапустите backend:
   ```bash
   ./velum.sh stop
   ./velum.sh start dev
   ```

4. Очистите кэш браузера (Ctrl+Shift+R / Cmd+Shift+R)
